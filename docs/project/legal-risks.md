# 法的リスク対策ドキュメント

**作成日**: 2025年12月
**更新日**: 2025年12月
**対象システム**: 教室運営支援システム

---

## 1. 概要

本ドキュメントは、習い事教室運営支援システム（特に子供バレエ教室向け）を運営する上で考慮すべき法的リスクとその対策をまとめたものです。

### 1.1 本システムの法的リスク要因

- **子供の個人情報を扱う**: 特に慎重な取り扱いが必要
- **位置情報・行動情報を含む**: いつ・誰が・どこにいるかの情報
- **LINE Platform連携**: 第三者サービスとの連携によるリスク
- **マルチテナント構成**: 複数事業者のデータを同一システムで管理
- **多言語・多国籍利用者**: 外国人利用者への配慮

## 2. 個人情報保護法対応

### 2.1 法的根拠

**個人情報の保護に関する法律（個人情報保護法）**
- 令和2年改正（令和4年4月1日全面施行）
- 令和3年改正（令和5年4月1日施行）

### 2.2 個人情報取扱事業者としての義務

#### 該当する個人情報

| データ種別 | 該当する個人情報 | 要配慮個人情報 |
|----------|--------------|-------------|
| 生徒情報 | 氏名、生年月日、学年 | ❌ |
| 保護者情報 | 氏名、LINE ID、連絡先 | ❌ |
| レッスン参加記録 | 出欠状況、参加履歴 | ❌ |
| 発表会情報 | キャスティング、役名 | ❌ |
| 健康情報（※） | 怪我・アレルギー情報 | ✅（要配慮個人情報）|

※ 将来的に健康情報を扱う場合は、明示的な同意が必要

#### 必須対応事項

1. **利用目的の明示**
   - システム利用開始時に利用目的を明確に通知
   - プライバシーポリシーの作成と公開
   - 変更時の通知義務

2. **適正取得**
   - 偽りその他不正の手段による取得の禁止
   - 第三者提供時の本人同意取得

3. **安全管理措置**
   - 技術的安全管理措置の実施（後述）
   - 従業者への監督
   - 委託先への監督（Cloudflare、LINEへの委託）

4. **本人の権利対応**
   - 開示請求への対応（1ヶ月以内）
   - 訂正・削除請求への対応
   - 利用停止請求への対応

### 2.3 子供の個人情報の特別な配慮

#### 法的要請

- **15歳未満の子供の個人情報**: 親権者の同意が必要（実務上の慣行）
- **EU GDPR**: 16歳未満は親権者の同意必須（EU居住者が対象の場合）
- **米国COPPA**: 13歳未満の子供のデータ収集に親の同意必須（米国在住者が対象の場合）

#### 実装対応

```
[必須]
✅ 保護者アカウントによる登録・管理
✅ 子供本人による直接登録の禁止
✅ 保護者の同意記録の保存

[推奨]
✅ 年齢層に応じた情報取得の最小化
✅ 子供の写真・動画は原則システム外で管理
✅ 子供のプライバシー保護方針の明記
```

### 2.4 プライバシーポリシー記載事項

#### 必須記載項目

1. **事業者の名称・連絡先**
   - テナント（各教室）の名称・代表者
   - システム提供者の名称・連絡先

2. **個人情報の利用目的**
   - レッスン運営管理
   - 出欠管理・連絡
   - 発表会準備管理
   - 保護者への情報提供

3. **取得する個人情報の項目**
   - 生徒: 氏名、生年月日、学年
   - 保護者: 氏名、LINE ID、連絡先
   - システム利用記録: アクセスログ、操作履歴

4. **第三者提供**
   - Cloudflare（システム基盤提供者）
   - LINE（メッセージング基盤提供者）
   - 第三者提供の法的根拠と停止手続き

5. **保有期間**
   - 生徒在籍中＋退会後1年間（推奨）
   - 財務記録: 7年間（税法上の保管義務）

6. **開示等の請求手続き**
   - 請求窓口の連絡先
   - 本人確認方法
   - 手数料の有無

7. **Cookie・アクセス解析**
   - 使用するCookie・トラッキング技術
   - 利用目的
   - 拒否方法

### 2.5 技術的安全管理措置

#### 実装済み対策

| 対策 | 実装内容 |
|-----|---------|
| アクセス制御 | LINE Login / パスワード認証による認証 |
| 権限管理 | owner / admin / guardian の3段階権限 |
| 通信の暗号化 | HTTPS通信（Cloudflare SSL/TLS） |
| データベース暗号化 | Cloudflare D1（保存時暗号化） |
| アクセスログ | access_log テーブルでの記録 |

#### 追加推奨対策

```
[推奨]
□ 2要素認証の提供（管理者向け）
□ IPアドレス制限（管理画面）
□ セッションタイムアウト設定
□ 異常アクセス検知・アラート
□ 定期的な脆弱性診断
□ バックアップとリストア手順の確立
```

## 4. LINE Platform利用規約遵守

### 4.1 LINE公式アカウント利用規約

#### 禁止事項

```
❌ スパム行為
❌ 利用者の同意なしの情報送信
❌ 第三者への不正なデータ提供
❌ LINE IDの不正取得
❌ 過度な頻度でのプッシュ通知
```

#### 本システムでの対応

```
✅ Reply Token活用による無料返信優先
✅ プッシュ通知は最小限（月200通制限）
✅ LINE IDは本人同意の上で取得
✅ 通知頻度の制限機能
```

### 4.2 LIFF利用規約

#### 重要事項

1. **利用者データの取り扱い**
   - LIFFアプリで取得したLINE User IDは、LINE以外のサービスと紐付けて利用可能
   - ただし、利用目的を明示し同意を得る必要あり

2. **プライバシーポリシーの掲示**
   - LIFFアプリ内でプライバシーポリシーへのリンクを設置

3. **データの第三者提供**
   - LINE User IDを第三者に提供する場合は同意が必要

### 4.3 LINE Messaging API利用規約

#### 送信頻度制限

```
[無料枠]
- Reply Token応答: 無制限
- プッシュ通知: 月200通まで

[本システムの方針]
✅ 通常運用はReply Token応答のみ
✅ プッシュ通知は緊急時のみ（臨時レッスン、休校）
✅ テナント設定で月間上限を管理
```

#### 禁止されるコンテンツ

```
❌ 暴力的・性的コンテンツ
❌ 差別的表現
❌ 違法行為の助長
❌ 第三者の権利侵害
```

## 5. 著作権・肖像権対応

### 5.1 音楽著作権

#### リスク

発表会で使用する楽曲の著作権侵害リスク

#### 対策

```
[システム側の方針]
✅ 楽曲データはシステムに保存しない
✅ 楽曲名のみテキストデータとして管理
✅ JASRAC等への申請は教室側の責任と明記

[利用規約への記載]
- 発表会での音楽利用は各教室の責任で適法に行うこと
- システムは楽曲管理機能を提供しない
```

### 5.2 肖像権

#### リスク

発表会の写真・動画における子供の肖像権

#### 対策

```
[システム設計]
✅ 写真・動画は原則システム外で管理
✅ システムには保存しない方針
✅ 外部サービス（Google Photos等）へのリンクのみ

[利用規約への記載]
- 写真・動画の撮影・公開には保護者の同意が必要
- システムは写真保管機能を提供しない
```

### 5.3 振付の著作権

#### リスク

バレエ振付の著作権（著作権法で保護される場合あり）

#### 対策

```
[システム側の方針]
✅ 振付の詳細な記録機能は提供しない
✅ 演目名、役名のみテキストで管理
✅ 指導内容の記録は教室側の責任

[教室向けガイドライン]
- 他者の振付を無断で複製・公開しないこと
- オリジナル振付の場合は著作権が発生する可能性を認識
```

### 8.1 サイバーセキュリティ基本法

#### 努力義務

```
[中小企業向けの推奨対応]
✅ 従業員へのセキュリティ教育
```

### 8.2 不正アクセス禁止法

#### 対応義務

```
[必須対応]
✅ アクセス制御機能（パスワード等）の設置
✅ ID・パスワードの適切な管理
✅ 不正アクセスの検知・防止

[禁止事項]
❌ 他人のIDを不正に取得・使用
❌ セキュリティホールを悪用
```

### 8.3 データ漏洩時の対応

#### 個人情報保護委員会への報告義務

```
[報告が必要な場合（令和4年改正）]
- 要配慮個人情報の漏えい
- 財産的被害が生じるおそれのある漏えい
- 1000人を超える個人情報の漏えい

[報告期限]
- 速報: 漏えい等を知った日から3〜5日以内
- 確報: 30日以内（調査完了後）
```

#### 本人への通知義務

```
[通知が必要な場合]
- 財産的被害が生じるおそれ
- 二次被害のおそれ

[通知内容]
1. 漏えい等の事実
2. 漏えい等した個人情報の項目
3. 原因
4. 二次被害防止のための措置
5. 問い合わせ窓口
```

#### インシデント対応フロー

```
1. 発見・認識
   ↓
2. 初動対応（被害拡大防止）
   ↓
3. 事実調査・原因究明
   ↓
4. 個人情報保護委員会への報告（該当する場合）
   ↓
5. 本人への通知（該当する場合）
   ↓
6. 再発防止策の実施
   ↓
7. 報告書作成・記録保存
```
